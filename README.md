# UI автотесты (VK Video + Alchemy)

## Описание проекта

В данном проекте реализованы UI автотесты для двух мобильных приложений:

- VK Video  
- Alchemy  

Тесты написаны с использованием Java, Appium и TestNG.

---

## Среда выполнения (Environment)

Тесты для Alchemy запускались на реальном Android-устройстве (Samsung S23).

Использование реального устройства обусловлено тем, что при работе с эмулятором наблюдалось нестабильное поведение:
- проблемы с запуском приложений
- некорректная работа рекламных сценариев

Тесты для VK Video запускались на эмуляторе.
  
---

## Настройка перед запуском

В классе `TestBase` предусмотрены настройки для двух приложений:

- VK Video (эмулятор)
- Alchemy (реальное устройство)

Для запуска тестов необходимо:

- раскомментировать блок настроек нужного приложения  
- закомментировать второй блок  

---

## Запуск тестов

Тесты запускаются из следующих классов:

- `VkVideoTests`
- `AlchemyTests`

Запуск возможен:
- через IntelliJ IDEA  
- через TestNG  

---

## Покрытие тестами

### VK Video

- Открытие видео из ленты  
- Проверка перехода на экран видео (по наличию кнопки Follow)  
- Негативный сценарий: нажатие вне карточки видео → экран не открывается  

---

### Alchemy

- Открытие экрана подсказок  
- Просмотр rewarded-рекламы  
- Обработка различных вариантов отображения рекламы (skip / close)  
- Проверка увеличения количества подсказок на 2  

---

## Особенности реализации

- Реализована обработка нескольких вариантов рекламы (разные локаторы кнопок закрытия)
- Используется цикл попыток для закрытия рекламы
- Добавлены ожидания для нестабильных элементов интерфейса

---
